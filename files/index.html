Servidor Utilizado

O servidor utilizado nesta box foi o Apache (apache2), pois, ele é o servidor mais popular e também é muito confiável. O apache é utilizado por gigantes como Netflix, Airbnb, eBay, Cisco, BBC, IBM, Nike, Microsoft, entre muitos são apenas alguns exemplos que usufruem das funcionalidades desse servidor que atende às solicitações de acesso de conteúdo de vários usuários simultaneamente.

Linguagem de provisionamento

A linguagem de provisionamento utiizada foi a Puppet, e foi escolhida pois é uma ferramenta de automação de TI open source para provisionamento, gerenciamento de configurações, implantação de aplicações, orquestraçãoe muitos outros processos manuais de TI.

Serviços provisionados

Foram provisionados 6 tipos de serviços, dos quais foram aplicados de forma simples e direta para cunho de estudo. Os serviços implementados são:
1. Samba
2. Proxy
3. Email
4. Firewall
5. Serviços de arquivos
6. Banco de dados

Linguagem de provisionamento - Samba

O primeiro serviço utilizado é o Samba, ele é um software livre que permite realizar a reimplementação do protocolo de rede SMB/CIFS. Embora tenha sido criado para integração com um domínio de servidor Windows, ele é uma boa opção para o Linux, e para a maioria dos sistemas operacionais do modelo Unix

Serviço provisionado - Proxy

O segundo serviço utilizado é o server proxy com o Squid, pois o uso do mesmo como Proxy/Cache é uma das opções mais utilizadas em ambientes corporativos que adotam software livre para administrar a rede, seja através da otimização e melhoria do desempenho da rede atuando como servidor de cache ou como proxy, implementando restrições de acesso à Internet, que nem sempre é possível fazer através do firewall.


O serviço provisionado foi um servidor proxy no qual seu objetivo é bloquear sites de buscas, tais quais:

  • google.com
  • bing.com
  • yahoo.com
  • duckduckgo.com
  • ask.com
  • aol.com
  • searchencrypt.com
  • msn.com
  • wolframalpha.com
  • nationmaster.com

Linguagem de provisionamento - Email

O terceito serviço utilizado é o Postfix , com ele é possível fazer a instalação de um webmail — uma caixa de e-mail que pode ser acessada com uso do navegador — chamado SquirrelMail, que se caracteriza por ser altamente leve e compatível com a maioria dos browsers, até mesmo os mais antigos.

Serviços provisionados - Firewall

O quarto serviço utilizado é o firewall com UFW, ele é um componente muito importante para a segurança de qualquer dispositivo que realiza conexões com a internet. Mesmo o Linux tendo ótimas ferramentas de segurança pré-instaladas, quanto mais seguro, melhor.

Linguagem de provisionamento - FTP

O quinto serviço utilizado é o FTP com VSFTPD, ele é um servidor FTP estável, seguro e rápido. É um dos mais conhecidos e utilizados, possuindo uma documentação muito acessível e que é de fácil acesso.

Serviços provisionados - Banco de dados

O sexto serviço utilizado é o banco de dados com PHP, é utilizado PHP pois é capaz de suportar grandes quantidades de dados, e com isso a linguagem consegue executar muitas funções e consumir muitos recursos ao mesmo tempo sem comprometer o desempenho e velocidade do servidor.

Vagrant File Used

Vagrant.configure("2") do |config| config.vm.define "server" do |server| server.vm.box = "ubuntu/bionic64" server.vm.network "private_network", ip: $ip server.vm.provider "virtualbox" do |virtualboxMachine| virtualboxMachine.name = "serverApacheAndPuppet" end end config.vm.provision :"shell", path: "installpuppet.sh" config.vm.provision :puppet do |puppet| puppet.manifests_path = "manifests" puppet.module_path = "modules" puppet.options = ['--verbose'] end config.vm.post_up_message = $message end
  1. Cole os arquivos em um novo diretório
    • VagrantFile
    • installpuppet.sh
    • /manifests
    • /modules
  2. Execute o seguinte comando no diretório
    • vagrant up
  3. Abra o navegador no link a seguir:192.168.60.58
